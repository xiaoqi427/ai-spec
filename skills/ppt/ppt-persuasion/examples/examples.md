# 使用示例

> PPT说服力大师（ppt-persuasion）技能的完整使用示例

---

## 示例1：Q3经营分析会汇报

### 用户输入

```bash
/skill ppt-persuasion plan --topic "2026年Q3经营分析报告" --audience "CEO及高管团队" --scene "季度经营分析会"
```

### AI 输出

```markdown
# PPT规划方案：《2026年Q3经营分析报告》

## 一、需求分析

| 项目 | 内容 |
|------|------|
| 主题 | 2026年Q3经营分析报告 |
| 目标 | 汇报Q3经营成果，获得Q4预算和战略方向审批 |
| 听众 | CEO及高管团队 |
| 场景 | 季度经营分析会 |
| 时长 | 25分钟 |
| 页数 | 22页 |
| 风格 | 商务简约、数据驱动 |

## 二、听众分析

**核心听众画像**：
- CEO：关注战略对齐和整体增长，偏好大局型，需要结论前置
- CFO：关注财务指标和成本控制，偏好数据型，需要详细ROI
- COO：关注运营效率和执行落地，偏好行动型，需要具体措施

**沟通策略**：
- 结论前置，3分钟内给出核心结论
- 用数据说话，每个论点都有数据支撑
- 坦诚面对问题，同时给出解决方案
- 在请求支持时，明确投入产出比

## 三、整体框架（22页）

| 页码 | 类型 | 标题 | 核心内容 |
|------|------|------|----------|
| 1 | 封面 | Q3营收超额达成115%，利润率环比提升2个百分点 | |
| 2 | 目录 | 四大板块导航 | |
| 3 | 过渡页 | 一、Q3经营成果 | |
| 4 | 数据看板 | 四大核心指标全面达标 | 营收/利润/客户/效率 |
| 5 | 内容页 | 营收：华东市场贡献最大增量 | 区域分布图 |
| 6 | 内容页 | 利润：成本优化项目带来2%利润率提升 | 成本瀑布图 |
| 7 | 内容页 | 客户：新增重点客户12家 | 客户增长趋势 |
| 8 | 过渡页 | 二、核心问题与挑战 | |
| 9 | 内容页 | 三大核心问题需要关注 | 问题清单（红黄绿） |
| 10 | 内容页 | 供应链效率影响交付周期 | 根因分析鱼骨图 |
| 11 | 内容页 | 人才缺口影响项目交付 | 人力数据对比 |
| 12 | 过渡页 | 三、Q4改进方案 | |
| 13 | 内容页 | 三大策略推动Q4突破 | 策略概览 |
| 14 | 内容页 | 策略一：供应链数字化改造 | 实施方案+时间表 |
| 15 | 内容页 | 策略二：关键岗位人才引进 | 招聘计划+预算 |
| 16 | 内容页 | 策略三：产品结构优化 | 产品矩阵调整 |
| 17 | 内容页 | Q4行动计划（甘特图） | 里程碑+责任人 |
| 18 | 过渡页 | 四、Q4目标与资源需求 | |
| 19 | 内容页 | Q4目标：营收5.5亿，利润率18% | SMART目标 |
| 20 | 内容页 | 资源需求：追加预算1200万 | 投入产出比分析 |

|------|------|------|

| 章节 | 时长 | 讲解重点 |
|------|------|----------|
| 经营成果 | 8min | 数据亮点，简明扼要 |
| 问题挑战 | 5min | 坦诚面对，不回避 |
| 改进方案 | 8min | 重点讲解，争取支持 |
| 目标资源 | 4min | 明确诉求，给出ROI |

## 六、说服力检查清单

- [x] 目标明确：获得Q4预算审批
- [x] 结论前置：封面即为核心结论
- [x] 数据支撑：每页有具体数据
- [x] 逻辑清晰：回顾→分析→方案→目标
- [x] 行动导向：明确的行动计划和资源请求
```

## 示例2：诊断PPT说服力

### 用户输入

```bash
/skill ppt-persuasion diagnose --file ./my-report-outline.md
```

### AI 输出

```markdown
# PPT说服力诊断报告

## 诊断结果：⚠️ 中等（需要改进）

### 金字塔得分

| 层级 | 状态 | 问题 |
|------|------|------|
| 观点鲜明 | ⚠️ | 标题是描述型而非观点型 |
| 逻辑清晰 | ❌ | 缺少明确的逻辑主线 |
| 言之有物 | ✅ | 数据充分 |
| 阅读友好 | ⚠️ | 部分页面文字过多 |
| 图文并茂 | ❌ | 图表使用不足 |
| 整洁美观 | ✅ | 风格统一 |
| 细节完美 | ⚠️ | 需要统一字号和间距 |

### 改进建议

1. **标题改写**（优先级：高）
   - 第3页 "销售数据" → "Q3销售额突破历史新高"
   - 第7页 "问题分析" → "供应链效率下降15%，根因在于..."

2. **逻辑重构**（优先级：高）
   - 建议采用"总分总"结构
   - 增加过渡页连接各章节

3. **文字精简**（优先级：中）
   - 第5页文字超过8行，建议精简至4行
   - 将详细说明放入附录

4. **增加图表**（优先级：中）
   - 第4页的数据表格改为柱状图
   - 第8页的对比文字改为矩阵图
```

## 示例3：单步执行（听众分析）

### 用户输入

```bash
/skill ppt-persuasion --step 2 --topic "AI赋能财务共享" --audience "CFO+各BU财务总监"
```

### AI 输出

```markdown
# 听众分析：《AI赋能财务共享》

## 核心听众画像

### 听众1：CFO（关键决策者）
- **角色特点**：全局视角，关注财务战略和风险
- **预判立场**：中立偏支持（关注ROI和实施风险）
- **核心痒点**：降本增效、数据驱动决策、合规风控
- **关注指标**：投资回报率、实施成本、见效周期
- **说服策略**：先讲ROI，再讲技术方案

### 听众2：各BU财务总监（影响者）
- **角色特点**：业务视角，关注实际操作影响
- **预判立场**：可能有顾虑（担心影响现有流程）
- **核心痒点**：减少重复工作、提高准确率、不增加负担
- **关注指标**：处理效率、错误率、培训成本
- **说服策略**：用标杆案例消除顾虑，强调"赋能而非替代"

## 异议预判

| 可能异议 | 来源 | 预设回应 |
|---------|------|---------|
| "投入太大" | CFO | 展示3年TCO对比和IRR |
| "会影响现有流程" | BU总监 | 展示渐进式实施方案 |
| "数据安全怎么保障" | CFO | 展示安全架构和合规认证 |
| "AI准确率够吗" | BU总监 | 展示POC验证数据 |

## 建议的说服路径

利益说服路径（适合务实型听众）：
现状痛点 → 不改变的代价 → AI带来的价值 → 投入产出比 → 实施计划
```
