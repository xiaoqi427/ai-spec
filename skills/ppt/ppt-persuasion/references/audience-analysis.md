# 听众分析模型与策略

> 来源：《说服力：让你的PPT会说话》 - 秋叶
> 核心理念：一份好PPT不在于提供了多少信息，而在于听众接受了多少

---

## 一、听众分析五维模型

### 维度1：找准核心听众

**问题**：在场的所有人中，谁是关键决策者？

| 场景 | 核心听众 | 策略 |
|------|----------|------|
| 预算审批会 | CFO / 财务总监 | 突出ROI和成本效益 |
| 项目汇报 | 直属上级 | 突出进度和风险 |
| 商业路演 | 投资人/合伙人 | 突出增长潜力和回报 |
| 产品评审 | 产品负责人 | 突出用户价值和竞争力 |
| 客户提案 | 客户决策者 | 突出价值和差异化 |

**行动**：列出在场者名单，圈出1-3位核心听众，围绕他们定制内容。

### 维度2：判断听众立场

| 立场 | 表现 | 策略 |
|------|------|------|
| **支持** | 已认同你的方向 | 强化信心，提供行动方案 |
| **中立** | 态度不明，观望中 | 提供充分证据，打消顾虑 |
| **反对** | 有不同意见或利益冲突 | 先认同其顾虑，再提供解决方案 |
| **无所谓** | 不关心此话题 | 找到与其利益的关联点 |

**行动**：对每位核心听众标注立场，针对性准备。

### 维度3：了解听众痒点

**问题**：听众最关心什么？什么能让他们兴奋？

| 听众角色 | 核心痒点 | PPT中的体现 |
|----------|----------|-------------|
| CEO/总经理 | 战略方向、业绩增长 | 宏观趋势、战略对齐、增长数据 |
| CFO | 财务回报、风险控制 | ROI分析、成本效益、风险评估 |
| CTO | 技术可行性、架构合理 | 技术方案、架构图、实施计划 |
| 销售负责人 | 市场机会、收入目标 | 市场数据、客户案例、销售预测 |
| 运营负责人 | 效率提升、流程优化 | 效率指标、流程对比、自动化方案 |
| HR负责人 | 人才发展、组织健康 | 团队建设、培训计划、人效数据 |

### 维度4：考虑部门立场

不同部门对同一件事有不同的利益诉求：

```
财务部门 ← 关注成本和ROI
         ↑
技术部门 ← 关注可行性和资源
         ↑
   你的方案
         ↓
业务部门 ← 关注效果和速度
         ↓
管理层   ← 关注战略对齐和风险
```

**行动**：提前预判各部门的可能反应，在PPT中预设回答。

### 维度5：考虑听众个性

| 类型 | 特征 | PPT策略 |
|------|------|---------|
| **数据型** | 注重数字和证据 | 多用数据图表，提供详细附录 |
| **故事型** | 喜欢案例和叙事 | 讲故事，用成功案例打动 |
| **细节型** | 关注执行细节 | 提供详细的行动计划和时间表 |
| **大局型** | 关注宏观方向 | 聚焦战略意义，简化细节 |
| **视觉型** | 喜欢图片和图表 | 用图说话，减少纯文字 |
| **行动型** | 关注下一步做什么 | 明确的行动项和责任人 |

## 二、听众画像模板

为每份PPT填写听众画像：

```markdown

- 姓名/角色：___
- 立场：支持 / 中立 / 反对

- 决策标准：___
- 可能的反对意见：___

### 次要听众
- 群体描述：___
- 共同特点：___
- 主要需求：___

### 沟通策略
- 开场切入点：___
- 核心说服路径：___
- 预设Q&A：___
```

## 三、场景-听众匹配矩阵

| 场景 | 典型听众 | 时长 | 深度 | 互动性 | 核心诉求 |
|------|----------|------|------|--------|----------|
| **董事会汇报** | 董事/高管 | 15-20min | 战略级 | 低 | 获得决策支持 |
| **经营分析会** | 部门负责人 | 20-30min | 运营级 | 中 | 分析问题、推动改进 |
| **项目启动会** | 项目团队 | 30-45min | 执行级 | 高 | 统一认知、分配任务 |
| **客户提案会** | 客户决策者 | 30-45min | 方案级 | 中 | 赢得合同 |
| **投资路演** | 投资人 | 15-20min | 商业级 | 中高 | 获得投资 |
| **培训分享会** | 学员 | 60-120min | 知识级 | 高 | 传授技能 |
| **年度总结会** | 全体员工 | 30-60min | 概览级 | 低 | 鼓舞士气 |

## 四、说服路径设计

### 理性说服路径（适合数据型听众）

```
事实（数据/现象）→ 分析（原因/逻辑）→ 结论（观点）→ 行动（方案）
```

### 感性说服路径（适合故事型听众）

```
共鸣（痛点/场景）→ 故事（案例/经历）→ 愿景（美好未来）→ 行动（如何实现）
```

### 权威说服路径（适合保守型听众）

```
权威背书（行业报告/专家观点）→ 标杆案例（成功实践）→ 方案论证 → 风险控制
```

### 利益说服路径（适合务实型听众）

```
现状（现有问题）→ 损失（不改变的代价）→ 收益（改变的价值）→ 行动（投入产出比）
```

## 五、异议预判与处理

### 异议预判模板

| 可能的反对意见 | 来源 | 预设回应 | PPT中的处理 |
|---------------|------|----------|-------------|
| "预算太高" | CFO | ROI分析 + 分阶段实施 | 单独一页展示投入产出比 |
| "时间不够" | 项目经理 | 分期实施计划 | 展示阶段性里程碑 |
| "技术风险大" | CTO | 技术方案 + 备选方案 | 附录中提供技术细节 |
| "市场不确定" | 业务负责人 | 市场调研数据 | 数据图表 + 案例支撑 |

### 预设 Q&A 准备

为每个可能的问题准备：
1. **直接回答**（30秒内）
2. **支撑数据**（1个关键数字）
3. **备用幻灯片**（附录中准备）

- [ ] 我知道核心听众是谁
- [ ] 我了解他们的立场（支持/中立/反对）
- [ ] 我知道他们最关心什么

- [ ] 我预判了可能的反对意见并准备了回应
- [ ] PPT的深度和风格适合目标听众
- [ ] 演示时长和节奏适合场景
- [ ] 我有备用方案应对突发提问
