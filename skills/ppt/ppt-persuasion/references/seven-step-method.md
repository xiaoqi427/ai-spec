# 七步法完整操作指南

> 来源：《说服力：让你的PPT会说话》 - 秋叶
> 核心思路：从目标出发，以听众为中心，系统化制作有说服力的PPT

---

## 步骤1：确定目标

### 核心原则
- **目标和手段要分开**：PPT是手段，不是目标本身
- **唯有内容吸引人**，才有可能达到真正的目标
- **目标是分层次的**，也是可以分阶段的

### SMART 目标原则

| 维度 | 含义 | 示例 |
|------|------|------|
| **S**pecific | 具体的 | "获得Q3项目预算200万审批" |
| **M**easurable | 可衡量的 | "预算通过率100%" |
| **A**vailable | 可达成的 | "基于现有数据和案例支撑" |
| **R**eliable | 相关的 | "与公司年度战略目标一致" |
| **T**ime-bound | 有时限的 | "本周五经营会上获批" |

### 麦肯锡方法论框架

```
现象 → 原因 → 对策 → 行动
```

**操作步骤**：
1. 用一句话写下PPT的核心目标
2. 用SMART原则检验目标是否合格
3. 确定成功标准：什么情况算"达到目的"
4. 识别关键障碍：听众可能的反对意见

### 输出物
- [ ] 一句话核心目标
- [ ] 成功标准定义
- [ ] 关键障碍列表

## 步骤2：分析听众

### 核心原则
- **一份好PPT不在于提供了多少信息，而在于听众接受了多少**
- **一定要有亮点**
- **没有最好的风格，只有最适合的风格**

### 听众分析五维模型

| 维度 | 关键问题 | 分析要点 |
|------|----------|----------|
| 核心听众 | 谁是关键决策者？ | 识别最重要的1-3个人 |
| 听众立场 | 对你的方案持什么态度？ | 支持/中立/反对 |
| 痒点需求 | 听众最关心什么？ | 业绩/成本/风险/创新 |
| 部门立场 | 各部门利益诉求？ | 财务关注ROI，技术关注可行性 |
| 个性特质 | 偏好什么表达方式？ | 数据型/故事型/细节型/大局型 |

### 听众类型适配策略

| 听众类型 | 特征 | PPT策略 |
|----------|------|---------|
| 决策者（老板/高管） | 时间少、关注结论 | 结论前置、数据精炼、突出价值 |
| 专业人士（技术/财务） | 重视细节和逻辑 | 数据充分、逻辑严密、附录详尽 |
| 业务伙伴（客户/合作方） | 关注利益和信任 | 案例驱动、收益可视化、建立信任 |
| 投资人 | 关注增长和回报 | 市场数据、商业模式、财务预测 |
| 学员/下属 | 需要学习和指导 | 循序渐进、互动参与、实操案例 |

### 场景适配

| 演示场合 | 时长 | 页数建议 | 风格建议 |
|----------|------|----------|----------|
| 电梯汇报 | 1-3分钟 | 3-5页 | 极简、结论前置 |
| 部门会议 | 10-15分钟 | 10-15页 | 清晰、数据驱动 |
| 经营分析会 | 20-30分钟 | 20-30页 | 专业、逻辑严密 |
| 商业路演 | 15-20分钟 | 15-20页 | 精美、故事化 |
| 培训演讲 | 60-120分钟 | 40-80页 | 图文并茂、互动性强 |

### 输出物
- [ ] 听众画像（角色、关注点、决策因素）
- [ ] 沟通策略（风格、深度、重点）
- [ ] 风格定位（配色方向、版式风格）

详见：[audience-analysis.md](audience-analysis.md)

## 步骤3：整体构思

### 核心原则
- **在PPT里讲一个好故事**
- **找一个联系所有材料的好线索**
- **整体构思前：关闭电脑、先做大纲、全局浏览**

### 故事线类型

| 线索类型 | 说明 | 适用场景 |
|----------|------|----------|
| 时间线 | 按时间顺序展开 | 项目汇报、发展历程 |
| 空间线 | 按地域/范围展开 | 市场分析、区域报告 |
| 逻辑线 | 按因果/递进展开 | 问题解决、方案论证 |
| 结构线 | 按维度/分类展开 | 产品介绍、能力展示 |
| 对比线 | 按前后/优劣展开 | 竞品分析、方案比选 |

### 标准PPT结构

```
┌─────────────────────────────┐
│ 封面页：主标题 + 副标题      │
├─────────────────────────────┤
│ 目录页：3-5个章节导航        │
├─────────────────────────────┤
│ 过渡页：章节标题 + 引导语    │ × N个章节
├─────────────────────────────┤
│ 内容页：一页一个核心观点      │ × 每章3-5页
├─────────────────────────────┤
│ 总结页：核心观点回顾          │
├─────────────────────────────┤
│ 致谢页：联系方式/Q&A         │
└─────────────────────────────┘
```

### SCQA 故事框架

| 要素 | 含义 | 示例 |
|------|------|------|
| **S**ituation | 情境 | "公司营收连续3个季度增长" |
| **C**omplication | 冲突 | "但利润率却在下降" |
| **Q**uestion | 问题 | "如何在保持增长的同时提升盈利能力？" |
| **A**nswer | 答案 | "三大策略：优化成本结构、提升客单价、拓展高毛利业务" |

### 输出物
- [ ] PPT大纲（章节名称和每章核心观点）
- [ ] 故事线设计（开头、发展、高潮、结尾）
- [ ] 页数分配方案

## 步骤4：组织材料

### 核心原则
- **挑重点说，知道取舍，而不是堆积**
- **用图说话**
- **把有限的版面留给重要的话**

### 素材提炼三标准

| 标准 | 说明 |
|------|------|
| 视觉冲击力 | 能否用图片/图表替代文字？ |
| 支撑主题 | 是否直接支持核心论点？ |
| 真实感 | 是否有数据/案例/证据？ |

### 逻辑组织方法

详见：[logic-frameworks.md](logic-frameworks.md)

1. **金字塔原理** - 结论先行，层层展开
2. **MECE法则** - 互不遗漏，互不重复
3. **归纳法** - 从具体到一般
4. **演绎法** - 从一般到具体

### 标题撰写原则

| 类型 | 差的标题 | 好的标题 |
|------|----------|----------|
| 描述型→观点型 | "销售数据分析" | "Q3销售额同比增长35%" |
| 模糊→具体 | "市场情况" | "华东市场占有率提升至28%" |
| 被动→主动 | "问题与挑战" | "三大核心问题亟需解决" |
| 冗长→精炼 | "关于下半年计划的一些想法" | "下半年三大突破方向" |

### 输出物
- [ ] 逐页内容规划（标题+要点+支撑数据）
- [ ] 数据可视化方案（图表类型选择）
- [ ] 素材清单（需要的图片/数据/案例）

## 步骤5：统一美化

### 核心原则
- **统一的PPT风格**
- **善用母版**
- **一开始就要确定页面风格**

### 风格统一检查项

| 检查项 | 要求 |
|--------|------|
| 配色方案 | 全篇使用统一色板（主色+辅色+强调色） |
| 字体系统 | 标题字体+正文字体，全篇不超过3种 |
| 版式模板 | 使用母版确保页面布局一致 |
| 图标风格 | 线性/填充/等距风格统一 |
| 图表样式 | 配色和样式与整体风格一致 |
| 间距规范 | 统一的页边距、行间距、元素间距 |

### 60-30-10 配色法则

```
60% 主色（背景/大面积区域）
30% 辅色（次要元素/图表）
10% 强调色（重点数据/关键信息）
```

### 场景配色建议

| 场景 | 推荐色系 | 情感传达 |
|------|----------|----------|
| 企业汇报 | 深蓝+灰白 | 专业、稳重、可信 |
| 科技创新 | 蓝紫+黑白 | 创新、前沿、科技感 |
| 金融财务 | 深蓝+金色 | 严谨、高端、信赖 |
| 教育培训 | 绿色+暖色 | 活力、温馨、亲和 |
| 品牌营销 | 品牌色+互补色 | 品牌识别、一致性 |

### 输出物
- [ ] 配色方案（HEX色值）
- [ ] 字体方案（标题/正文/数据）
- [ ] 母版设计规范

## 步骤6：单页设计

### 六大设计原则

| 原则 | 说明 | 操作要点 |
|------|------|----------|
| **对齐** | 元素间建立视觉联系 | 使用参考线，保持对齐 |
| **分离** | 相关信息靠近，无关信息拉远 | 用间距分组 |
| **留白** | 适当空间提升可读性 | 页面不超过60%填充 |
| **降噪** | 减少干扰元素 | 颜色不超过4种 |
| **重复** | 统一风格建立一致性 | 使用统一的版式 |
| **差异** | 强烈对比突出重点 | 大小/颜色/粗细对比 |

### 版式类型选择

| 版式 | 适用内容 | 特点 |
|------|----------|------|
| 左右对称 | 对比、并列关系 | 平衡、稳重 |
| 上下结构 | 层级、递进关系 | 逻辑清晰 |
| 居中对称 | 封面、核心观点 | 聚焦、有力 |
| 九宫格 | 多要素展示 | 全面、有序 |
| F型布局 | 信息阅读 | 符合阅读习惯 |

### 文字规范

| 元素 | 字号范围 | 说明 |
|------|----------|------|
| 主标题 | 36-48pt | 一眼能看到 |
| 副标题 | 24-32pt | 层级清晰 |
| 正文 | 18-24pt | 后排可读 |
| 注释 | 12-14pt | 辅助信息 |

### 动画建议
- **尽量不用动画**（秋叶原则）
- 如必须使用，选择简单的淡入/淡出
- 动画要服务于内容逻辑，不要喧宾夺主

### 输出物
- [ ] 逐页设计方案（版式+内容+视觉元素）
- [ ] 关键页面设计指导

## 步骤7：内部测试

### 检查清单

**内容层面**：
- [ ] 核心目标是否清晰传达
- [ ] 逻辑链条是否完整
- [ ] 数据是否准确
- [ ] 有无错别字和语法错误

**设计层面**：
- [ ] 风格是否全篇统一
- [ ] 对齐是否精确
- [ ] 配色是否协调
- [ ] 字号是否合适（后排可读性）

**技术层面**：
- [ ] 在目标设备上是否正常显示
- [ ] 字体是否嵌入或通用
- [ ] 动画是否正常播放
- [ ] 链接和跳转是否有效

**演示层面**：
- [ ] 预演时间是否在目标范围内
- [ ] 翻页逻辑是否顺畅
- [ ] 准备了PDF备份版本

### 兼容性建议
- 将所有页面转换成图片，再制作成PPT（无动画但兼容性最好）
- 转为PDF作为备份
- 提前到演示场地测试

### 输出物
- [ ] 检查清单完成状态
- [ ] 优化建议列表
- [ ] 最终版本确认
