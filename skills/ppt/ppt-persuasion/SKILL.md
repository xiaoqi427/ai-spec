# PPT说服力大师 (基于《说服力：让你的PPT会说话》)

> 作者：秋叶
> 核心理念：技巧服务于形式，形式服务于内容，内容服务于主题，主题服务于听众

## 使用方式

### 完整规划（推荐）

```bash
# 使用七步法完整规划一份PPT
/skill ppt-persuasion plan --topic <主题> --audience <听众> --scene <场景>

# 示例
/skill ppt-persuasion plan --topic "2026年度经营分析报告" --audience "高管团队" --scene "季度经营会"
```

### 单步执行

```bash
# 执行七步法中的某一步
/skill ppt-persuasion --step <步骤号> --topic <主题>
```

**示例**:
```bash
# 只做听众分析（步骤2）
/skill ppt-persuasion --step 2 --topic "产品路演" --audience "投资人"

# 只做逻辑梳理（步骤4）
/skill ppt-persuasion --step 4 --topic "项目汇报"
```

### 诊断优化

```bash
# 诊断现有PPT的说服力问题
/skill ppt-persuasion diagnose --file <PPT文件或大纲>

# 示例
/skill ppt-persuasion diagnose --file ./quarterly-report.md
```

### 参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 命令 | ✅ | plan / diagnose / --step | `plan` |
| --topic | ✅ | PPT主题 | `"年度总结"` |
| --audience | ❌ | 目标听众 | `"高管"`, `"客户"`, `"投资人"` |
| --scene | ❌ | 使用场景 | `"会议汇报"`, `"路演"`, `"培训"` |
| --style | ❌ | 风格偏好 | `"商务"`, `"简约"`, `"科技"` |
| --pages | ❌ | 预计页数 | `20`, `30` |
| --step | ❌ | 指定七步法步骤(1-7) | `1`, `3`, `6` |
| --file | ❌ | 现有PPT文件或大纲路径 | `./report.md` |

## 核心方法论：七步法

本技能基于秋叶《说服力》提出的 **"七步法"** 系统化方法论：

| 步骤 | 名称 | 核心任务 | 详细文档 |
|------|------|----------|----------|
| 1 | 确定目标 | 明确PPT的核心目的和预期成果 | [步骤详解](references/seven-step-method.md#步骤1确定目标) |
| 2 | 分析听众 | 研究听众需求、立场和痛点 | [听众分析](references/audience-analysis.md) |
| 3 | 整体构思 | 构建故事线和整体框架 | [步骤详解](references/seven-step-method.md#步骤3整体构思) |
| 4 | 组织材料 | 逻辑梳理、素材提炼 | [逻辑框架](references/logic-frameworks.md) |
| 5 | 统一美化 | 确定整体视觉风格 | [步骤详解](references/seven-step-method.md#步骤5统一美化) |
| 6 | 单页设计 | 逐页设计和优化 | [步骤详解](references/seven-step-method.md#步骤6单页设计) |
| 7 | 内部测试 | 预演、检查和修正 | [步骤详解](references/seven-step-method.md#步骤7内部测试) |

## 执行流程

### 第一阶段：需求理解

AI 根据用户输入，提取并确认以下信息：

| 信息项 | 来源 | 示例 |
|--------|------|------|
| 主题 | 命令参数 / 询问 | "2026年度经营分析" |
| 核心目标 | 询问用户 | "获得预算审批" |
| 目标听众 | 命令参数 / 询问 | "CFO及财务团队" |
| 使用场景 | 命令参数 / 询问 | "季度经营分析会" |
| 预计时长 | 询问用户 | "20分钟" |
| 页数范围 | 命令参数 / 自动估算 | "15-20页" |
| 风格偏好 | 命令参数 / 推荐 | "商务简约" |

### 第二阶段：结构设计（七步法执行）

**严格按照七步法顺序执行**：

1. **确定目标** → 明确核心诉求和成功标准
2. **分析听众** → 听众画像、关注点、决策因素
3. **整体构思** → 故事线设计、大纲框架
4. **组织材料** → 逻辑结构、素材安排、数据可视化
5. **统一美化** → 配色方案、字体系统、版式规范
6. **单页设计** → 逐页内容和视觉方案
7. **内部测试** → 检查清单和优化建议

### 第三阶段：输出交付

## PPT说服力金字塔

《说服力》提出的PPT质量层级模型（从底层到顶层）：

```
        ┌──────────┐
        │ 细节完美  │  ← 第7层：完美呈现
       ┌┴──────────┴┐
       │  整洁美观   │  ← 第6层：视觉专业
      ┌┴────────────┴┐
      │  图文并茂    │  ← 第5层：视觉表达
     ┌┴──────────────┴┐
     │  阅读友好      │  ← 第4层：信息可读
    ┌┴────────────────┴┐
    │   言之有物       │  ← 第3层：内容充实
   ┌┴──────────────────┴┐
   │    逻辑清晰        │  ← 第2层：结构严谨
  ┌┴────────────────────┴┐
  │     观点鲜明         │  ← 第1层：核心明确
  └──────────────────────┘
```

**关键认知**：
- Power Point = 有 Power 的 Point
- 技巧服务于形式，形式服务于内容，内容服务于主题，主题服务于听众
- 工作型PPT首推"达到目的"，不过度追求美感
- 用情感打动你，用逻辑说服你

## 核心原则

### 1. 目标导向
- 每份PPT必须有明确的、可衡量的目标
- 目标和手段要分开，PPT是手段不是目标
- 使用 SMART 原则检验目标设定

### 2. 听众至上
- 一份好PPT不在于提供了多少信息，而在于听众接受了多少
- 给观众选择题或判断题，而不是问答题
- 没有最好的风格，只有最适合的风格

### 3. 逻辑优先
- 高手培养的是结构化表达复杂问题的逻辑思维能力
- 提供核心观点，而非堆积大量数据
- 使用金字塔原理组织内容

### 4. 内容为王
- 首先提供有价值的内容，其次要美观
- 挑重点说，知道取舍，而不是堆积
- 把有限的版面留给重要的话

### 5. 讲好故事
- 在PPT里讲一个好故事
- 找一个联系所有材料的好线索
- 有开头、发展、高潮、结尾

## 六大设计原则

1. **对齐** - 元素间建立视觉联系
2. **分离** - 无关信息距离拉大，相关信息靠近
3. **留白** - 适当空间提升可读性
4. **降噪** - 颜色不要太多，减少干扰元素
5. **重复** - 统一风格，建立一致性
6. **差异** - 强烈对比，突出重点

## 学习路径

秋叶推荐的学习方法：

1. **向营销界学定位** - 精准定义PPT的核心价值主张
2. **向咨询师学逻辑** - 用金字塔原理和MECE法则组织内容
3. **向广告圈学文案** - 写出有冲击力的标题和文案
4. **向设计师学排版** - 掌握视觉设计的基本原则
5. **向培训师学表达** - 提升演讲和互动技巧

**行动三步曲**：
- 从模仿开始行动
- 从微调开始创新
- 从灵感开始创意

## 强制约束

### 说服力检查清单

每份PPT完成后必须通过以下检查：

- [ ] **目标明确**：能一句话说清这份PPT要达到什么目的
- [ ] **听众适配**：内容深度和风格适合目标听众
- [ ] **观点鲜明**：每页只传达一个核心观点
- [ ] **逻辑清晰**：整体结构经得起"所以呢？"的追问
- [ ] **言之有物**：数据和案例支撑每个论点
- [ ] **标题有力**：每页标题是观点而非描述
- [ ] **图文并茂**：用图表代替大段文字
- [ ] **风格统一**：配色、字体、版式全篇一致
- [ ] **留白适当**：页面不拥挤，信息层级清晰
- [ ] **可以演示**：在目标设备上测试无兼容性问题
