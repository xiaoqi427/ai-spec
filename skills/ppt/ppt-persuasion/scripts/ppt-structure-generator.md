# PPT结构自动生成指南

> 本文档描述 AI 如何根据用户输入自动生成 PPT 结构规划

---

## 使用方式

当用户触发 `ppt-persuasion` 技能时，AI 按照以下流程自动生成 PPT 结构。

---

## 输入收集

### 必填信息

```yaml
topic: ""          # PPT主题
purpose: ""        # 核心目的（一句话）
audience: ""       # 目标听众
scene: ""          # 使用场景
```

### 可选信息

```yaml
duration: ""       # 演示时长（分钟）
pages: ""          # 期望页数
style: ""          # 风格偏好（商务/简约/科技/创意）
brand_color: ""    # 品牌色（HEX）
key_points: []     # 核心内容要点
data_sources: []   # 可用的数据来源
constraints: []    # 特殊限制（如"不能超过15页"）
```

---

## 生成流程

### Step 1：信息补全

根据用户提供的信息，推断缺失项：

```
IF 未提供时长:
  根据场景推断默认时长
  会议汇报 → 20min
  商业路演 → 15min
  培训教学 → 60min

IF 未提供页数:
  页数 = 时长(min) × 1页/min（粗略估算）
  
IF 未提供风格:
  根据听众和场景推荐风格
```

### Step 2：选择框架模板

根据场景匹配模板：

| 场景关键词 | 匹配模板 |
|-----------|----------|
| 汇报、总结、周报、月报 | business-report-template |
| 路演、融资、商业计划 | roadshow-template |
| 培训、教学、分享、workshop | training-template |
| 其他 | 通用框架（自动组装） |

### Step 3：生成大纲

根据七步法生成完整的 PPT 大纲结构：

```markdown
## PPT大纲：《{topic}》

### 基本信息
- 主题：{topic}
- 目标：{purpose}
- 听众：{audience}
- 场景：{scene}
- 时长：{duration}分钟
- 页数：{pages}页
- 风格：{style}

### 听众分析
- 核心听众画像：{根据输入生成}
- 听众关注点：{根据角色推断}
- 说服策略：{根据立场推断}

### 结构框架
{根据模板生成逐页大纲}

### 配色方案
- 主色：{根据风格推荐}
- 辅色：{根据主色推荐}
- 强调色：{根据主色推荐}

### 字体方案
- 标题：{推荐}
- 正文：{推荐}

### 注意事项
{根据场景生成}
```

### Step 4：逐页细化

对大纲中的每一页提供详细设计方案：

```markdown
### 第{N}页：{页面标题}

**核心观点**：{一句话}

**内容要点**：
1. {要点1}
2. {要点2}
3. {要点3}

**版式建议**：{左右/上下/居中/九宫格}

**视觉元素**：
- 图表：{类型}
- 图标：{描述}
- 图片：{风格}

**设计提示**：{特别注意事项}
```

## 输出格式

### 完整输出模板

```markdown
# PPT规划方案：《{主题}》

## 一、需求分析
| 项目 | 内容 |
|------|------|
| 主题 | {topic} |
| 目标 | {purpose} |
| 听众 | {audience} |
| 场景 | {scene} |
| 时长 | {duration} |
| 页数 | {pages} |
| 风格 | {style} |

## 二、听众分析
{听众画像和沟通策略}

## 三、整体框架
{大纲结构表}

## 四、逐页设计方案
{每页详细设计}

## 五、配色与字体
{视觉规范}

## 六、演示建议
{时间分配和讲解重点}

## 七、检查清单
{说服力检查项}
```

## 场景速查

| 用户说 | AI 动作 |
|--------|---------|
| "帮我做一个汇报PPT" | 使用 business-report-template，询问缺失信息 |
| "路演PPT怎么做" | 使用 roadshow-template，引导 SCQA 故事线 |
| "做一个培训课件" | 使用 training-template，确认知识模块 |
| "优化这份PPT" | 使用诊断模式，检查说服力金字塔 |
| "帮我梳理PPT逻辑" | 使用逻辑框架，检查结构 |
