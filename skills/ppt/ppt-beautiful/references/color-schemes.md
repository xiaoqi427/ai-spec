## 一、配色基础理论

### 60-30-10 配色法则

```
┌────────────────────────────────────────┐
│                                        │
│              60% 主色                   │  ← 背景/大面积区域
│         （奠定整体基调）                 │
│                                        │
├─────────────────────┬──────────────────┤
│                     │                  │
│     30% 辅色        │                  │  ← 次要元素/标题
│  （丰富视觉层次）    │   10% 强调色     │  ← 重点数据/CTA
│                     │ （突出关键信息）   │
│                     │                  │
└─────────────────────┴──────────────────┘
```

### 配色方案构成

```yaml
完整配色方案:
  主色: "#XXXXXX"     # 大面积使用，定义基调
  辅色: "#XXXXXX"     # 辅助区分，丰富层次
  强调色: "#XXXXXX"   # 突出重点，吸引注意
  背景色: "#XXXXXX"   # 页面背景
  文字主色: "#XXXXXX" # 正文文字
  文字副色: "#XXXXXX" # 辅助文字
  正向色: "#XXXXXX"   # 正面/增长/达标
  负向色: "#XXXXXX"   # 负面/下降/未达标
```

## 三、配色禁忌

### 绝对避免

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| 颜色过多 | 超过5种主要颜色 | 控制在3-4种 |
| 纯色叠加 | 红配绿、蓝配橙 | 使用同色系 |
| 饱和度失控 | 全部使用高饱和色 | 降低辅色饱和度 |
| 对比不足 | 浅色文字配浅色背景 | 确保对比度≥4.5:1 |
| 风格不统一 | 前面商务蓝后面活力橙 | 全篇一套色板 |

### 安全配色建议

1. **不确定时选蓝色** - 蓝色是最安全的商务色
2. **用灰色做缓冲** - 不同颜色间用灰色过渡
3. **强调色面积要小** - 强调色只用在关键数据上
4. **深色浅色搭配** - 避免两个深色或两个浅色放一起
5. **参考品牌色** - 优先使用公司/客户的品牌色

## 四、从品牌色生成配色方案

当用户提供品牌色时，按以下规则生成完整配色：

```
输入：品牌色（如 #1890FF）

生成规则：
├── 主色 = 品牌色
├── 辅色 = 品牌色降低饱和度20%，提亮10%
├── 强调色 = 品牌色的互补色（色轮对面），或品牌色暖色方向偏移30°
├── 背景色 = 品牌色极浅版本（亮度95%+）或纯白
├── 文字主色 = #1A202C 至 #2D3748（深灰/近黑）
└── 文字副色 = #718096 至 #A0AEC0（中灰）
```

## 五、数据可视化配色

### 图表配色规则

| 数据类型 | 配色策略 |
|----------|----------|
| 单系列 | 使用主色，关键数据用强调色 |
| 多系列 | 主色→辅色→强调色，同色系深浅变化 |
| 正负对比 | 正向=#38A169(绿)，负向=#E53E3E(红) |
| 占比数据 | 主体=主色，其余=灰色或浅色 |
| 高亮重点 | 目标数据=强调色，其余=灰色 |

### 图表灰化技巧

**突出一个数据的方法**：
- 目标数据保持鲜艳颜色
- 其他数据全部变为浅灰色 #E2E8F0
- 视觉上自然聚焦到关键数据
