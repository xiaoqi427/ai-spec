# PPT视觉设计大师 (基于《PPT，要你好看》)

> 作者：邵云蛟
> 核心理念：三大设计思维（整体性、结构性、可视化）+ 视觉风格公式 + 四步高级排版

## 使用方式

### 视觉设计（推荐）

```bash
# 为PPT提供完整的视觉设计方案
/skill ppt-beautiful design --topic <主题> --style <风格> --pages <页数>

# 示例
/skill ppt-beautiful design --topic "年度总结" --style "科技蓝" --pages 20
```

### 单页美化

```bash
# 美化某一页的排版和视觉
/skill ppt-beautiful page --type <页面类型> --content <内容描述>

# 示例
/skill ppt-beautiful page --type "封面" --content "2026年度财务共享中心工作总结"
/skill ppt-beautiful page --type "数据页" --content "四个核心KPI指标展示"
```

### 配色方案

```bash
# 获取配色方案推荐
/skill ppt-beautiful color --scene <场景> --mood <情感>

# 示例
/skill ppt-beautiful color --scene "商务汇报" --mood "专业稳重"
```

### 设计诊断

```bash
# 诊断PPT的视觉设计问题
/skill ppt-beautiful check --file <文件或描述>
```

### 参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 命令 | ✅ | design / page / color / check | `design` |
| --topic | ❌ | PPT主题 | `"年度总结"` |
| --style | ❌ | 视觉风格 | `"商务蓝"`, `"科技紫"`, `"简约白"` |
| --pages | ❌ | 页数 | `20` |
| --type | ❌ | 页面类型 | `"封面"`, `"目录"`, `"数据页"` |
| --content | ❌ | 页面内容描述 | `"三个并列要点"` |
| --scene | ❌ | 使用场景 | `"商务汇报"`, `"产品发布"` |
| --mood | ❌ | 情感调性 | `"专业"`, `"活力"`, `"高端"` |
| --brand | ❌ | 品牌色 | `"#1890FF"` |

## 核心方法论

### 三大设计思维

| 思维 | 核心含义 | 操作要点 |
|------|----------|----------|
| **整体性思维** | PPT是一个整体，每页都是系统的一部分 | 统一风格、统一色彩、统一字体、统一版式 |
| **结构性思维** | 信息需要有层次和结构 | 标题层级、内容分区、视觉引导线 |
| **可视化思维** | 能用图不用表，能用表不用文 | 图表优先、图标辅助、减少纯文字 |

### 视觉风格公式

```
视觉风格 = 配色方案 + 字体组合 + 版式规范 + 装饰元素
```

**四大要素缺一不可**：
1. **配色方案** → 决定PPT的情感调性
2. **字体组合** → 决定PPT的专业程度
3. **版式规范** → 决定PPT的信息可读性
4. **装饰元素** → 决定PPT的视觉记忆点

### 四步高级排版技巧

| 步骤 | 名称 | 核心操作 |
|------|------|----------|
| Step 1 | **整理** | 提取关键信息，删减冗余内容 |
| Step 2 | **对齐** | 元素对齐，建立视觉秩序 |
| Step 3 | **聚拢** | 相关信息靠近，形成视觉组 |
| Step 4 | **精调** | 对比、留白、层级微调 |

## 执行流程

### 第一阶段：风格定位

根据用户需求确定视觉风格：

| 信息项 | 来源 | 作用 |
|--------|------|------|
| 主题/行业 | 用户输入 | 确定风格方向 |
| 使用场景 | 用户输入 | 确定正式程度 |
| 品牌色 | 用户提供/推荐 | 确定配色基础 |
| 情感调性 | 用户偏好/推荐 | 确定视觉气质 |
| 参考风格 | 用户示意/推荐 | 确定设计方向 |

### 第二阶段：方案输出

根据视觉风格公式输出完整方案：

1. **配色方案** → 主色+辅色+强调色+背景色+文字色（详见 [配色方案](references/color-schemes.md)）
2. **字体方案** → 标题字体+正文字体+数据字体（详见 [字体排版](references/typography-guide.md)）
3. **版式规范** → 母版设计+页面网格+间距规范
4. **装饰元素** → 图标风格+装饰线条+背景处理

### 第三阶段：逐页设计

根据页面类型提供具体的排版方案，详见 [templates/](templates/) 目录：

## 设计原则（来自《PPT，要你好看》）

### 1. 一致性原则
- 全篇使用统一的配色方案
- 全篇使用统一的字体组合（不超过3种）
- 统一的图标风格（线性/填充/等距）
- 统一的间距和留白规范

### 2. 对比原则
- **大小对比**：标题大、正文小、注释更小
- **颜色对比**：重点信息用强调色
- **粗细对比**：标题加粗、正文常规
- **疏密对比**：重要区域留白多、次要区域紧凑

### 3. 亲密性原则
- 相关元素靠近放置
- 不相关元素拉开距离
- 用间距而非线条分组
- 视觉上形成自然的信息模块

### 4. 对齐原则
- 每个元素都要有对齐参考
- 左对齐是最安全的选择
- 居中对齐适合少量文字
- 使用参考线确保精确

### 5. 留白原则
- 页面不要填满，留足呼吸空间
- 页边距统一（建议左右各2cm）
- 元素之间留足间距
- 留白也是设计的一部分

### 6. 降噪原则
- 去掉不必要的装饰线条
- 去掉多余的图表网格线
- 减少颜色种类（不超过4种主要色）
- 减少字体种类（不超过3种）

## Before-After 设计优化原则

《PPT，要你好看》核心方法——通过 Before-After 对比学习设计：

### 优化维度

| 维度 | Before（问题） | After（改进） |
|------|---------------|--------------|
| 文字 | 大段文字堆砌 | 提炼关键词+图标辅助 |
| 配色 | 颜色杂乱无章 | 统一色板，3-4种颜色 |
| 排版 | 元素随意摆放 | 网格对齐，留白充足 |
| 图表 | 默认Excel图表 | 简化元素，突出数据 |
| 图片 | 低质量/水印图 | 高清大图+蒙版处理 |
| 图标 | 风格不统一 | 统一线性/填充风格 |

## 强制约束

### 视觉设计检查清单

每份PPT完成后必须通过以下检查：

- [ ] **配色统一**：全篇使用同一套色板
- [ ] **字体统一**：不超过3种字体
- [ ] **对齐精确**：所有元素有对齐参考
- [ ] **留白充足**：页面填充不超过70%
- [ ] **层级清晰**：标题>副标题>正文>注释
- [ ] **图标一致**：统一线性或填充风格
- [ ] **图片高清**：无水印、无变形、无拉伸
- [ ] **图表简洁**：去除多余网格线和装饰
- [ ] **颜色克制**：主要颜色不超过4种
- [ ] **间距一致**：相同类型的间距相等
