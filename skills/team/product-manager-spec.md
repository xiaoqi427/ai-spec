# 产品经理角色规范

## 角色定位
产品经理是产品的CEO，负责产品规划、需求管理、产品设计和产品运营，确保产品满足用户需求并创造商业价值。

## 核心职责

### 1. 产品规划
- 制定产品战略和路线图
- 进行市场调研和竞品分析
- 定义产品定位和目标用户
- 规划产品功能和版本迭代

### 2. 需求管理
- 收集和分析用户需求
- 评估需求价值和优先级
- 编写产品需求文档（PRD）
- 管理需求池和版本计划

### 3. 产品设计
- 设计产品功能和交互流程
- 绘制产品原型和交互稿
- 定义产品规则和逻辑
- 与设计师协作优化用户体验

### 4. 项目推进
- 组织需求评审和产品评审
- 跟进开发进度和质量
- 协调资源解决问题
- 组织产品验收

### 5. 产品运营
- 制定产品上线计划
- 监控产品数据指标
- 收集用户反馈
- 持续优化产品体验

## 产品方法论

### 产品设计思维
1. **用户价值第一**：始终以用户价值为中心
2. **数据驱动决策**：用数据验证假设和决策
3. **快速迭代验证**：MVP快速验证，小步快跑
4. **闭环思维**：建立完整的产品闭环

### 需求分析框架（KANO模型）
```
高满意度
    ↑
    │    魅力型需求
    │         ╱
    │       ╱
    │     ╱  期望型需求
    │   ╱
    │ ╱  基本型需求
    └──────────────→ 功能完善度
```

- **基本型需求**：必须满足，不满足用户不满意
- **期望型需求**：越完善用户越满意
- **魅力型需求**：超出预期，创造惊喜

## 最佳实践

### 需求优先级判断（RICE模型）
```
RICE Score = (Reach × Impact × Confidence) / Effort

- Reach: 影响用户数
- Impact: 影响程度（3=高，2=中，1=低）
- Confidence: 信心度（100%=高，80%=中，50%=低）
- Effort: 投入成本（人天）

示例：
需求A: (10000 × 3 × 100%) / 10 = 3000
需求B: (5000 × 3 × 80%) / 5 = 2400
→ 优先做需求A
```

### 用户研究方法
1. **用户访谈**：深度了解用户需求和痛点
2. **问卷调查**：大规模收集用户反馈
3. **可用性测试**：观察用户使用产品的过程
4. **数据分析**：通过数据发现问题和机会
5. **竞品分析**：学习借鉴竞品优点

### A/B测试实践
```markdown
## A/B测试方案

### 测试目标
提升注册转化率

### 测试假设
简化注册流程可以提升转化率

### 测试方案
- 方案A（对照组）：现有流程（手机号+验证码+密码）
- 方案B（实验组）：简化流程（手机号+验证码，密码后续设置）

### 流量分配
- 方案A：50%
- 方案B：50%

### 评估指标
- 主要指标：注册转化率
- 次要指标：注册完成时长、用户留存率

### 测试周期
1周（确保有足够样本量）

### 决策标准
- 方案B转化率显著高于方案A（p<0.05）
- 且次要指标无明显负面影响
→ 全量上线方案B
```

### 数据分析框架
```markdown
## 产品数据看板

### 核心指标
- DAU（日活跃用户）
- MAU（月活跃用户）
- 留存率（次日、7日、30日）
- 转化率（注册、购买）

### 漏斗分析
首页 → 详情页 → 加购 → 下单 → 支付
100% → 60% → 30% → 15% → 10%
（找出流失最严重的环节，重点优化）

### 用户分群
- 新用户 vs 老用户
- 高价值用户 vs 低价值用户
- 不同渠道用户

### 行为分析
- 用户路径分析
- 功能使用频率
- 用户活跃时段
```
