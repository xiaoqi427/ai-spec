# 企业连续体（Enterprise Continuum）

> 企业连续体是 TOGAF 用于对架构资产进行分类的框架，
> 从最通用的基础架构到最具体的组织特定架构，形成一个连续的光谱。

---

## 1. 核心概念

企业连续体提供了一种对可复用架构资产进行分类和组织的方法。

```
基础架构 ──→ 通用系统架构 ──→ 行业架构 ──→ 组织特定架构
(最通用)                                    (最具体)
```

### 两个组成部分

```
企业连续体
├── 架构连续体（Architecture Continuum）
│   ├── 基础架构 ─ 最通用的架构模式和原则
│   ├── 通用系统架构 ─ 跨行业的通用系统模式
│   ├── 行业架构 ─ 特定行业的架构模式
│   └── 组织架构 ─ 组织自身的具体架构
│
└── 解决方案连续体（Solutions Continuum）
    ├── 基础解决方案 ─ 通用技术产品和框架
    ├── 通用系统解决方案 ─ 跨行业通用组件
    ├── 行业解决方案 ─ 行业特定产品和平台
    └── 组织解决方案 ─ 组织定制的实现
```

---

## 2. 架构连续体详解

### 2.1 基础架构（Foundation Architecture）

最抽象、最通用的架构概念和模式。

**来源**: TOGAF TRM（技术参考模型）、开放标准

**示例**:
- 分层架构模式
- 微服务架构模式
- 事件驱动架构模式
- RESTful API 设计原则
- 安全架构基本框架（认证/授权/加密）
- 数据管理基本原则（ACID/BASE/CAP）

**价值**: 提供跨组织、跨行业可复用的基本架构指导

### 2.2 通用系统架构（Common Systems Architecture）

适用于多个行业的通用系统模式。

**示例**:
- ERP 系统架构模式
- CRM 系统架构模式
- 统一身份认证架构
- 消息中间件架构
- 日志和监控架构
- API 网关架构

**价值**: 减少重复设计，复用跨行业最佳实践

### 2.3 行业架构（Industry Architecture）

特定行业的架构模式和参考架构。

**示例（金融/财务行业）**:
- 财务共享服务中心架构
- 核心银行系统架构
- 支付清算系统架构
- 风控系统架构
- 合规报告系统架构

**示例（制造行业）**:
- 供应链管理系统架构
- 制造执行系统（MES）架构
- 质量管理系统架构

**价值**: 利用行业沉淀的架构经验，加速项目交付

### 2.4 组织特定架构（Organization-Specific Architecture）

针对特定组织的具体架构。

**示例（FSSC 项目）**:
- FSSC 报账系统架构
- FSSC 微服务拆分方案
- FSSC 数据集成架构
- FSSC 审批流程引擎架构

**价值**: 直接指导组织的架构实施

## 3. 解决方案连续体详解

解决方案连续体与架构连续体一一对应，但关注具体的技术实现。

| 架构连续体 | 解决方案连续体 | 示例 |
|-----------|---------------|------|
| 基础架构 | 基础解决方案 | Linux, TCP/IP, HTTP |
| 通用系统架构 | 通用系统解决方案 | Spring Boot, MySQL, Redis |
| 行业架构 | 行业解决方案 | 财务共享平台, ERP 套件 |
| 组织架构 | 组织解决方案 | FSSC 微服务集群 |

## 4. 在 ADM 中的应用

### 4.1 资产复用流程

```
1. 识别需求 → 明确需要什么架构能力
2. 搜索连续体 → 从通用到具体查找可复用资产
   ├── 先查组织特定架构 ─ 最适合但最少
   ├── 再查行业架构 ─ 行业最佳实践
   ├── 再查通用系统架构 ─ 跨行业通用方案
   └── 最后查基础架构 ─ 最通用的模式
3. 评估适配度 → 判断资产是否可直接复用或需要定制
4. 定制使用 → 将通用资产定制为组织特定实现
5. 回馈归档 → 将新成果归入架构存储库
```

### 4.2 各 ADM 阶段的复用点

| ADM 阶段 | 可复用资产 | 复用方式 |
|----------|-----------|---------|
| 预备 | 架构原则模板 | 参考基础架构原则，定制组织原则 |
| A | 架构愿景模板 | 参考行业愿景模板 |
| B | 业务能力模型 | 参考行业业务能力框架 |
| C | 数据模型/应用模式 | 参考通用系统架构 |
| D | 技术参考模型 | 参考 TOGAF TRM |
| E-F | 迁移模式 | 参考行业迁移经验 |
| G | 治理框架 | 参考通用治理模式 |

## 5. 架构存储库与企业连续体

企业连续体的资产存储在架构存储库中：

```
架构存储库
├── 架构元模型 ─ 定义架构元素和关系
├── 参考库
│   ├── 基础架构参考 ─ TRM, 标准模式
│   ├── 通用系统参考 ─ 通用技术组件
│   └── 行业架构参考 ─ 行业参考模型
├── 组织架构资产
│   ├── 架构原则（AP）
│   ├── 架构愿景（AV）
│   ├── 架构定义（AD）
│   ├── 架构路线图（AR）
│   └── 架构决策（ADR）
├── 标准信息库
│   ├── TRM（技术参考模型）
│   ├── SIB（标准信息库）
│   └── 合规要求
├── 构建块库
│   ├── ABB（架构构建块）
│   └── SBB（解决方案构建块）
└── 治理日志
    ├── 合规评审（CR）
    ├── 变更记录（CHG）
    └── 架构合同（AC）
```

## 6. 实践建议

### 6.1 建设架构资产库

1. **从项目中提取** — 每个项目结束后，将可复用的架构模式归档
2. **分级管理** — 按连续体的四个层级分类管理
3. **版本控制** — 架构资产需要版本管理
4. **定期评审** — 定期审查资产的有效性和适用性

### 6.2 促进资产复用

1. **先搜索后创建** — 在设计新架构前，先搜索已有资产
2. **建立搜索机制** — 让架构师能方便地搜索和发现可复用资产
3. **奖励复用** — 从组织层面鼓励架构资产的复用
4. **降低复用门槛** — 提供清晰的文档和使用指南

### 6.3 FSSC 项目的企业连续体应用

| 层级 | FSSC 可用资产 |
|------|-------------|
| 基础架构 | 微服务模式、REST API 规范、安全框架 |
| 通用系统 | Spring Cloud 架构、MyBatis Plus 模式、Redis 缓存策略 |
| 行业架构 | 财务共享服务参考模型、报账流程标准 |
| 组织架构 | FSSC 分层架构规范、模块划分标准 |
