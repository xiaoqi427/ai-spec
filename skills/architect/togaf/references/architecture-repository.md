# 架构存储库（Architecture Repository）

> 架构存储库是组织架构资产的中央管理库，
> 存储架构原则、决策、构建块、标准和治理记录等所有架构相关资产。

---

## 1. 存储库结构

```
Architecture Repository
├── 架构元模型
│   ├── 元模型定义 ─ 架构元素类型和关系
│   └── 视点定义 ─ 面向不同利益相关者的视图
│
├── 参考库（Reference Library）
│   ├── 基础架构参考 ─ TOGAF TRM, 标准模式, 最佳实践
│   ├── 通用系统参考 ─ 跨行业通用架构组件
│   └── 行业架构参考 ─ 行业参考模型和标准
│
├── 组织架构资产（Organization Architecture Assets）
│   ├── 架构原则（AP）─ 指导架构决策的基本原则
│   ├── 架构愿景（AV）─ 目标架构的高层描述
│   ├── 架构定义（AD）─ 四域架构的详细设计
│   ├── 架构路线图（AR）─ 实施时间表和工作包
│   └── 架构决策（ADR）─ 重要架构决策的记录
│
├── 标准信息库（Standards Information Base）
│   ├── TRM（技术参考模型）─ 分层技术组件参考
│   ├── SIB（标准信息库）─ 批准使用的标准清单
│   └── 合规要求 ─ 内外部合规要求
│
├── 构建块库（Building Blocks Library）
│   ├── ABB（架构构建块）─ 能力定义
│   └── SBB（解决方案构建块）─ 实现方案
│
├── 治理日志（Governance Log）
│   ├── 合规评审（CR）─ 合规性评审记录
│   ├── 变更记录（CHG）─ 架构变更记录
│   └── 架构合同（AC）─ 实施约束协议
│
└── 需求库（Requirements Repository）
    ├── 业务需求 ─ 业务层面的需求
    ├── 架构需求 ─ 架构层面的需求
    └── 变更请求 ─ 架构变更请求
```

## 2. 各组成部分详解

### 2.1 架构元模型

定义了架构中使用的所有元素类型及其关系。

**核心元素类型**:
| 元素类型 | 描述 | 示例 |
|----------|------|------|
| 业务能力 | 组织的业务能力 | 报账能力、审批能力 |
| 业务服务 | 业务能力的服务化 | 报账单提交服务 |
| 数据实体 | 核心数据对象 | 报账单、发票 |
| 应用组件 | 应用或服务 | fssc-claim-service |
| 技术组件 | 技术基础设施 | MySQL, Redis, RabbitMQ |
| 架构原则 | 指导性原则 | 服务松耦合原则 |
| 构建块 | 可复用架构单元 | ABB/SBB |

### 2.2 参考库

存储可复用的参考架构和模式。

**组织方式**:
```
参考库/
├── patterns/           # 架构模式
│   ├── microservice/   # 微服务模式
│   ├── event-driven/   # 事件驱动模式
│   └── layered/        # 分层架构模式
├── standards/          # 行业标准
│   ├── iso/            # ISO 标准
│   └── togaf/          # TOGAF 标准
└── best-practices/     # 最佳实践
    ├── security/       # 安全最佳实践
    └── performance/    # 性能最佳实践
```

### 2.3 标准信息库（SIB）

记录组织批准使用的技术标准。

**技术标准目录示例**:

| 类别 | 标准项 | 批准版本 | 状态 | 生效日期 |
|------|--------|---------|------|----------|
| 编程语言 | Java | 21 | 当前 | 2024-01-01 |
| Web框架 | Spring Boot | 3.x | 当前 | 2024-01-01 |
| 微服务 | Spring Cloud | 2023.x | 当前 | 2024-01-01 |
| 数据库 | MySQL | 8.x | 当前 | 2024-01-01 |
| 缓存 | Redis | 7.x | 当前 | 2024-01-01 |
| 消息队列 | RabbitMQ | 3.x | 当前 | 2024-01-01 |
| ORM | MyBatis Plus | 3.5.x | 当前 | 2024-01-01 |
| 对象映射 | MapStruct | 1.5.x | 当前 | 2024-01-01 |
| 构建工具 | Maven | 3.9.x | 当前 | 2024-01-01 |

**标准状态**:
- **当前**: 正在使用，推荐
- **新兴**: 计划引入，试用中
- **过渡**: 正在淘汰，不推荐新项目使用
- **废弃**: 已停止使用

## 4. 版本管理

### 4.1 版本号规则

采用语义版本号 `v[Major].[Minor].[Patch]`:
- **Major**: 重大变更（架构原则变更、破坏性修改）
- **Minor**: 功能性变更（新增构建块、新增标准）
- **Patch**: 修正性变更（文档修正、描述优化）

### 4.2 变更记录

每个资产都应该记录变更历史：

| 版本 | 日期 | 变更人 | 变更说明 |
|------|------|--------|----------|
| v1.0 | [...] | [...] | 初始版本 |
| v1.1 | [...] | [...] | [...变更说明] |
| v2.0 | [...] | [...] | [...重大变更说明] |

## 7. 维护策略

1. **定期审查** — 每季度审查一次架构资产的有效性
2. **及时归档** — 项目完成后 2 周内归档架构资产
3. **清理废弃** — 标记废弃资产，保留历史但不在活跃目录
4. **质量把控** — 新资产入库需要经过评审
5. **培训推广** — 定期培训架构团队使用存储库
