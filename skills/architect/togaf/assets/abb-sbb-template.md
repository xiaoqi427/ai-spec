## 三、ABB 到 SBB 的映射关系

```
ABB（架构构建块）          SBB（解决方案构建块）
┌──────────────┐         ┌──────────────────┐
│ 需要什么能力   │  ──→   │ 用什么技术实现     │
│ 功能需求      │  ──→   │ 具体产品/框架     │
│ 非功能需求    │  ──→   │ 性能配置/架构设计  │
│ 接口规范      │  ──→   │ API 实现/消息协议  │
└──────────────┘         └──────────────────┘
```

**设计原则**:
1. **一个 ABB 可对应多个 SBB** — 同一能力可以有不同的实现方案
2. **先定义 ABB，再设计 SBB** — 确保实现方案满足能力需求
3. **ABB 稳定，SBB 可替换** — 技术选型变更只影响 SBB
4. **构建块可复用** — 设计时考虑跨项目复用性
