### 填写指引

1. **需求描述**：使用"作为...我需要...以便..."的用户故事格式
2. **验收条件**：必须具体、可测试，不能写"功能正常"
3. **追溯关系**：至少关联到一个业务目标和一个业务事件
4. **优先级**：P0 = 系统不可用级别，P1 = 核心业务流程，P2 = 重要功能，P3 = 优化项
5. **状态流转**：草稿 → 待确认 → 已确认 → 开发中 → 已实现 → 已验证
