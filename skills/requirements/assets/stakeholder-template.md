### 2. 权力-利益矩阵

```
            高利益
            │
   重点管理  │  密切合作
   (高权力   │  (高权力
    低利益)  │   高利益)
─────────────┼─────────────── 高权力
   最少关注  │  保持知情
   (低权力   │  (低权力
    低利益)  │   高利益)
            │
            低利益
```

| 象限 | 干系人 | 管理策略 |
|------|--------|---------|
| 密切合作（高权力高利益） | [SH-XX] | 主动管理，频繁沟通，优先满足需求 |
| 重点管理（高权力低利益） | [SH-XX] | 保持满意，定期汇报，关键决策时参与 |
| 保持知情（低权力高利益） | [SH-XX] | 充分沟通，及时通知变更 |
| 最少关注（低权力低利益） | [SH-XX] | 监控，必要时沟通 |

### 5. 填写指引

1. **识别范围**：不仅包括直接用户，还包括决策者、监管方、受影响方
2. **权力-利益评估**：客观评估，不同象限采用不同管理策略
3. **需求冲突**：当不同干系人的需求冲突时，以高权力高利益方的需求为优先
4. **定期更新**：干系人分析应随项目推进定期更新
