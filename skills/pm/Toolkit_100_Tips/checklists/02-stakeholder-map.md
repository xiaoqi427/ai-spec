# T02 干系人权力地图 检查清单

> **适用场景**: 项目启动时识别关键干系人，了解其权力和利益格局。
> **核心原则**: 搞清楚"谁说了算、谁关心、谁受影响"。

---

## 操作步骤

1. 列出所有可能的干系人（至少覆盖：决策者、出资人、使用者、执行者、影响者）
2. 评估每个人的权力等级（高/中/低）和利益等级（高/中/低）
3. 放入权力-利益四象限
4. 为每个象限制定差异化的沟通策略
5. 标注潜在的联盟和冲突关系
6. 制定重点干系人的一对一沟通计划

---

## 检查清单

- [ ] 已识别项目发起人及其期望
- [ ] 已识别最终用户/客户代表
- [ ] 已识别提供资源的职能经理
- [ ] 已识别可能反对项目的人
- [ ] 已识别可以影响项目成败的"幕后人物"
- [ ] 每个干系人的权力和利益都已评估
- [ ] 干系人已放入四象限图
- [ ] 高权力高利益的干系人有一对一沟通计划
- [ ] 高权力低利益的干系人有定期汇报计划
- [ ] 潜在的冲突关系已标注
- [ ] 干系人地图已与项目团队核心成员分享

---

## 快速分类提示

| 问自己这些问题 | 如果"是" → 分类 |
|-------------|---------------|
| 此人能否叫停项目？ | → 高权力 |
| 此人的日常工作受项目直接影响？ | → 高利益 |
| 此人控制项目所需的关键资源？ | → 高权力 |
| 此人是项目交付物的直接使用者？ | → 高利益 |
| 此人有否决权或审批权？ | → 高权力 |
