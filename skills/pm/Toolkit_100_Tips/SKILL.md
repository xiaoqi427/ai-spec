# 项目管理工具箱 — 100 个实战技巧

> 基于《项目管理工具箱》，提供面向具体问题的 100 个实战技巧与检查清单。
> **核心理念：项目管理不是纸上谈兵，而是在复杂约束中用对的工具解决对的问题。**

---

## 使用方式

### 1. 问题检索

```bash
# 根据问题描述检索匹配的技巧
/skill toolkit-100-tips search <问题描述>

# 按分类浏览技巧列表
/skill toolkit-100-tips list --category <分类>

# 查看特定编号的技巧详情
/skill toolkit-100-tips tip <编号>
```

**示例**:
```bash
/skill toolkit-100-tips search "项目进度一直延误怎么办"
/skill toolkit-100-tips list --category "进度管理"
/skill toolkit-100-tips tip 42
```

### 2. 检查清单

```bash
# 获取某个阶段的检查清单
/skill toolkit-100-tips checklist <阶段/主题>

# 获取某个具体场景的操作清单
/skill toolkit-100-tips checklist --scene <场景>
```

**示例**:
```bash
/skill toolkit-100-tips checklist "项目启动"
/skill toolkit-100-tips checklist --scene "项目复盘会"
```

### 3. 情景诊断

```bash
# 描述当前困境，获取诊断建议和对应技巧
/skill toolkit-100-tips diagnose <困境描述>
```

**示例**:
```bash
/skill toolkit-100-tips diagnose "团队成员积极性不高，每次开会都沉默"
```

### 参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 命令 | ✅ | search / list / tip / checklist / diagnose | `search` |
| --category | ❌ | 技巧分类 | `"启动"`, `"规划"`, `"执行"`, `"监控"`, `"收尾"`, `"领导力"`, `"沟通"` |
| --scene | ❌ | 具体场景 | `"项目启动会"`, `"需求评审"`, `"进度赶工"` |

## 执行流程

```
1. 问题识别 → 用户描述具体问题或场景
2. 关键词匹配 → 从问题中提取关键词，匹配技巧分类
3. 技巧推荐 → 推荐 1-3 个最相关的技巧
4. 详情展示 → 展示技巧的具体操作步骤和检查清单
5. 场景适配 → 根据用户的实际场景裁剪建议
```

### 响应级别

| 级别 | 触发条件 | 输出形式 |
|------|---------|---------|
| **快速** | 查看特定技巧 | 技巧卡片（名称+步骤+清单） |
| **标准** | 问题检索/场景诊断 | 2-3个推荐技巧 + 操作指南 |
| **深度** | 复杂困境诊断 | 完整分析 + 多技巧组合方案 |

---

## 强制约束

**必须做**：
1. 推荐的技巧必须贴合用户的实际问题，不要泛泛推荐
2. 每个技巧都要包含可操作的具体步骤
3. 信息不足时先追问场景细节
4. 用中文回答，工具名称附英文原文

**不要做**：
1. 不要一次推荐过多技巧（最多3个，聚焦最相关的）
2. 不要只给理论不给操作步骤
3. 不要忽视用户的项目规模和行业背景
4. 不要生搬硬套，要根据实际情况灵活调整
