# WBS 工作分解结构模板 (Work Breakdown Structure)

> **用途**: 将项目可交付成果和项目工作分解为较小的、更易于管理的组件。
> **所属过程**: 规划过程组 → 范围管理 → 创建 WBS

## WBS 分解原则

1. **100% 规则**: WBS 必须包含项目范围内的全部工作，且仅包含项目范围内的工作
2. **可交付成果导向**: 每个节点代表一个可交付成果，而非活动
3. **8/80 规则**: 最底层工作包的工作量在 8-80 小时之间
4. **互斥性**: 同一层级的元素之间没有重叠
5. **可管理性**: 每个工作包可以分配给一个责任人

## 使用说明

1. **第一步**: 确认项目范围说明书已经批准
2. **第二步**: 识别主要可交付成果（通常按阶段或功能模块分解）
3. **第三步**: 逐层分解直到满足 8/80 规则
4. **第四步**: 为每个工作包编写 WBS 词典条目
5. **第五步**: 经项目团队和关键干系人评审确认
