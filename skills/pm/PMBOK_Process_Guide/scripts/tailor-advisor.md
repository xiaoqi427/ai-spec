# 过程裁剪顾问 (Tailor Advisor)

> 根据项目类型、规模和开发方法，智能推荐 PMBOK 过程裁剪方案。

---

## 裁剪维度

### 按项目规模裁剪

| 过程 | 大型项目 | 中型项目 | 小型项目 |
|------|:------:|:------:|:------:|
| 制定项目章程 | ✅ 完整版 | ✅ 精简版 | ✅ 一页纸 |
| 制定项目管理计划 | ✅ 完整版 | ✅ 精简版 | ⚡ 简要清单 |
| 创建WBS | ✅ 多层分解 | ✅ 2-3层 | ⚡ 任务清单 |
| 识别风险 | ✅ 完整分析 | ✅ Top10风险 | ⚡ Top5风险 |
| 规划沟通管理 | ✅ 完整矩阵 | ✅ 关键沟通 | ⚡ 口头约定 |
| 规划采购管理 | ✅ 如需要 | ✅ 如需要 | ⚡ 按需 |
| 干系人登记册 | ✅ 完整版 | ✅ 关键干系人 | ⚡ 简要清单 |
| 变更控制 | ✅ 正式CCB | ✅ 简化流程 | ⚡ PM审批 |
| 经验教训 | ✅ 持续记录 | ✅ 阶段总结 | ⚡ 项目结束 |

**图例**: ✅ 推荐执行 | ⚡ 可简化执行 | ❌ 可跳过

### 按开发方法裁剪

| 过程 | 瀑布(Waterfall) | 敏捷(Agile) | 混合(Hybrid) |
|------|:------:|:------:|:------:|
| 项目章程 | ✅ 前期完成 | ✅ 精益画布 | ✅ 精简版 |
| 范围管理 | ✅ 前期固定 | 🔄 迭代演进 | 🔄 核心固定+迭代 |
| WBS | ✅ 完整分解 | 🔄 产品待办列表 | 🔄 高层WBS+待办 |
| 进度管理 | ✅ 甘特图 | 🔄 迭代计划 | 🔄 里程碑+迭代 |
| 风险管理 | ✅ 正式风险分析 | 🔄 迭代风险评审 | ✅ 正式+迭代 |
| 质量管理 | ✅ 质量审计 | 🔄 持续集成/TDD | 🔄 混合 |
| 变更控制 | ✅ 正式CCB | 🔄 产品负责人决策 | 🔄 分级控制 |

**图例**: ✅ 传统方式 | 🔄 适应型方式

### 按项目类型裁剪

| 项目类型 | 重点过程 | 可简化过程 |
|---------|---------|-----------|
| **IT系统开发** | 范围管理、风险管理、质量管理、变更控制 | 采购管理（如无外包） |
| **基建工程** | 进度管理、成本管理、采购管理、资源管理 | — |
| **产品研发** | 范围管理（需求迭代）、风险管理、干系人管理 | 采购管理 |
| **组织变革** | 干系人管理、沟通管理、风险管理 | 采购管理、成本管理 |
| **运维升级** | 变更控制、风险管理、质量管理 | 范围管理（范围较固定） |

---

## 裁剪决策流程

```
1. 确定项目规模 → 选择文档详细程度
2. 确定开发方法 → 选择过程执行方式
3. 确定项目类型 → 选择重点知识领域
4. 组合裁剪建议 → 生成裁剪后的过程清单
5. 团队评审确认 → 形成项目管理方法论
```

---

## 使用方式

AI 在接收到 `tailor` 命令时：

1. 解析用户提供的项目特征（类型、规模、方法）
2. 在三个裁剪维度中分别匹配推荐方案
3. 综合生成裁剪建议清单
4. 为每个推荐/简化的过程附上推荐的模板或替代工具
