# 过程映射引擎 (Process Mapper)

> 将用户描述的业务活动自动映射到 PMBOK 标准过程，提供过程组、知识领域和关键工具建议。

---

## 映射规则

### 信号词 → 过程组映射

| 信号词/关键词 | 映射过程组 | 关键过程 |
|-------------|-----------|---------|
| "启动""立项""授权""章程""可行性" | 启动 | 制定项目章程、识别干系人 |
| "规划""计划""WBS""预算""进度""风险识别" | 规划 | 对应知识领域的规划过程 |
| "执行""开发""实施""采购""团队建设" | 执行 | 指导与管理项目工作 |
| "监控""偏差""挣值""变更""审计""质量控制" | 监控 | 监控项目工作、实施整体变更控制 |
| "收尾""验收""交付""移交""经验教训" | 收尾 | 结束项目或阶段 |

### 信号词 → 知识领域映射

| 信号词/关键词 | 映射知识领域 |
|-------------|------------|
| "范围""需求""WBS""交付物""验收" | 范围管理 |
| "进度""关键路径""甘特图""里程碑""工期" | 进度管理 |
| "成本""预算""挣值""EVM""储备" | 成本管理 |
| "质量""审计""控制图""检查""测试" | 质量管理 |
| "资源""团队""RACI""冲突""培训" | 资源管理 |
| "沟通""会议""报告""干系人""信息分发" | 沟通管理 |
| "风险""概率""影响""应对""蒙特卡洛" | 风险管理 |
| "采购""合同""招标""供应商""SOW" | 采购管理 |
| "干系人""权力""利益""参与度""期望" | 干系人管理 |
| "变更""整合""章程""收尾""经验教训" | 整合管理 |

---

## 使用方式

AI 在接收到用户的项目管理相关问题时，按以下步骤执行过程映射：

1. **提取关键词**: 从用户描述中提取信号词
2. **匹配过程组**: 根据信号词确定所属过程组
3. **匹配知识领域**: 根据信号词确定涉及的知识领域
4. **定位具体过程**: 在过程组×知识领域的交叉点找到具体过程
5. **推荐工具与模板**: 提供该过程对应的工具、技术和模板

---

## 映射示例

**用户输入**: "我们需要评估项目的可行性并获得管理层批准"

**映射结果**:
- 过程组: **启动**
- 知识领域: **整合管理**
- 具体过程: **制定项目章程**
- 推荐模板: [project-charter.md](../templates/project-charter.md)
- 关键工具: 专家判断、数据收集（头脑风暴、焦点小组）、商业论证
