# FinMind 工作流：五阶段闭环

## 标准执行流程

当用户请求分析时（如 `/skill finmind analyze 600519 --framework value`），系统按以下五阶段闭环执行。

---

## 阶段1: 信号感知（Observer 观察智能体）

**触发**: 用户输入标的代码

**执行步骤**:

1. **解析请求参数**
   - 标的代码 → 识别市场（A股/美股/港股）
   - 分析框架 → value / trend / both
   - 报告类型 → deep / brief

2. **路径A: AI工具采集实时数据**
   ```
   search_web → 搜索"{标的名称} 最新行情 财务数据"
   fetch_content → 抓取东方财富个股页面 / 雪球个股页面
   ```

3. **路径B: Python脚本采集历史数据**（如需要）
   ```bash
   python scripts/fetch_market_data.py --symbol 600519 --period 1y
   python scripts/fetch_financials.py --symbol 600519
   ```

4. **数据标准化输出**
   ```
   ✅ 行情数据: 最新价/涨跌幅/成交量/换手率
   ✅ 财务数据: PE/PB/ROE/营收增速/净利增速/现金流
   ✅ 新闻舆情: 近7日重要新闻列表
   ✅ 资金流向: 北向/融资/主力资金
   ```

**输出格式**:
```
【阶段1: 数据采集完成】

📊 标的: 贵州茅台(600519) | A股
📅 数据时效: 截至 2026-03-15

行情概览:
- 最新价: ¥1,856.00 | 涨跌: +2.3%
- 成交额: 48.2亿 | 换手率: 0.38%
- 52周范围: ¥1,420 ~ ¥2,050

关键财务指标:
- PE(TTM): 28.5 | PB: 9.2
- ROE: 32.1% | 净利率: 51.3%
- 营收增速: 15.8% | 净利增速: 18.2%

近期要闻:
- [3/14] 茅台一季度批价稳定，渠道库存低位
- [3/12] 白酒行业政策利好，消费复苏预期升温
- [3/10] 外资连续5日净买入茅台

→ 进入阶段2: 深度分析
```

---

## 阶段2: 深度研究（Analyst 分析智能体）

**根据框架选择不同分析路径**:

### 路径A: 价值投资框架（--framework value）

依次执行价值投资五步法：

```
Step 1. 能力圈判断 → 我理解这门生意吗?
Step 2. 商业模式评估 → 护城河/管理层/行业空间
Step 3. 财务健康度检验 → 三表分析/杜邦分析
Step 4. 估值与安全边际 → DCF/PE/PB + 安全边际
Step 5. 芒格检查清单 → 25项逐一检查
```

**调用技能**:
- `fundamentals-parser` → 财报深度解析
- `factor-scoring-engine` → 多因子评分（价值/成长/质量维度）

**输出**: 价值评分卡（5维度 × 10分 = 50分满分）

### 路径B: 趋势交易框架（--framework trend）

依次执行趋势交易四象限：

```
Quadrant 1. 宏观周期定位 → 康波/朱格拉/基钦
Quadrant 2. 市场情绪测量 → 恐贪指数/资金流/关键位
Quadrant 3. 技术形态研判 → 趋势/形态/量价/关键位
Quadrant 4. 交易计划制定 → 入场/止损/止盈/仓位
```

**调用技能**:
- `kondratieff-cycle-analyzer` → 周期定位
- `sentiment-analyzer` → 情绪评分
- `technical-pattern-recognizer` → 形态识别
- `factor-scoring-engine` → 动量因子

**输出**: 趋势信号卡（买入/卖出/观望 + 关键位）

### 路径C: 双框架交叉验证（--framework both）

- 并行执行路径A和路径B
- 分别输出评分卡和信号卡
- 标注两套框架的**共识点**和**分歧点**

**输出格式**:
```
【阶段2: 分析完成】

📈 价值框架评分:
  能力圈: 9/10 | 商业模式: 8/10 | 财务健康: 8/10
  估值: 6/10 | 检查清单: 8/10
  ➤ 综合评分: 39/50 (优秀)

📊 趋势框架信号:
  周期定位: 繁荣后期 | 情绪: 偏乐观(0.65)
  技术形态: 突破前高 | 量价: 放量上攻
  ➤ 信号: 短期看多，注意高位风险

🔄 框架交叉:
  共识: 基本面优秀，当前趋势向上
  分歧: 价值框架认为估值偏高，趋势框架认为动量充足
  
→ 进入阶段3: 偏差检测
```

---

## 阶段3: 偏差检测（强制环节，不可跳过）

**核心目的**: 防止认知偏差影响决策质量。

**执行步骤**:

1. **认知偏差扫描**（引用 checklists/bias-detection.md）
   - 锚定效应: 是否被某个价格/数字锚定?
   - 确认偏差: 是否只看到了支持自己观点的证据?
   - 损失厌恶: 是否因为已有持仓而不愿承认错误?
   - 从众效应: 是否因为市场热度而盲目跟随?
   - 过度自信: 对分析结论的置信度是否合理?
   - 近因偏差: 是否过度关注近期数据而忽略长期趋势?

2. **芒格反转检验**
   - "什么情况下这笔投资会让我亏大钱?"
   - 列出至少3个风险场景

3. **段永平三不原则检验**
   - 不做空: ✅/❌
   - 不借钱投资: ✅/❌
   - 不做不懂的: ✅/❌

**输出格式**:
```
【阶段3: 偏差检测】

🧠 认知偏差扫描:
  ✅ 锚定效应: 未检测到
  ⚠️ 确认偏差: 轻微 — 分析中偏重正面信息，建议补充看空观点
  ✅ 损失厌恶: 不适用（新标的）
  ✅ 从众效应: 未检测到
  ✅ 过度自信: 置信度0.78，合理范围
  ✅ 近因偏差: 已包含3年历史数据

🔄 芒格反转检验:
  风险场景1: 白酒消费税政策调整 → 影响净利率3-5个百分点
  风险场景2: 宏观经济下行导致高端消费萎缩 → 营收增速降至个位数
  风险场景3: 管理层换届带来战略不确定性

✅ 段永平三不原则: 全部通过

→ 进入阶段4: 决策输出
```

---

## 阶段4: 决策输出（Strategist 决策智能体）

**执行步骤**:

1. **综合评分汇总**
   - 价值评分: XX/50
   - 趋势信号: 多/空/中性
   - 偏差检测: 通过/有风险

2. **决策矩阵**
   ```
   评分 ≥ 35 且 偏差检测通过 → 可以买入
   评分 25-34 → 观察，等待更好时机
   评分 < 25 → 不符合标准，PASS
   
   趋势信号为"多" → 增加仓位权重
   趋势信号为"空" → 降低仓位或等待
   ```

3. **仓位建议计算**
   - 基于凯利公式的简化版
   - 考虑当前组合风险水平
   - 段永平式集中 vs 达利欧式分散的平衡

4. **风控预检查**
   - 调用 risk-guardrail 检查仓位上限
   - 检查单笔风险敞口
   - 检查组合相关性

**输出格式**:
```
【阶段4: 决策建议】

🎯 最终建议: 买入
📊 置信度: 78%
💰 建议仓位: 15% (约 ¥XX万)

价格规划:
  入场区间: ¥1,830 ~ ¥1,870
  止损位: ¥1,720 (-7.3%)
  目标位1: ¥2,050 (+10.5%)
  目标位2: ¥2,200 (+18.5%)

决策依据:
  ✅ 价值评分 39/50 > 阈值35 → 基本面优秀
  ✅ 趋势信号看多 → 技术面配合
  ✅ 偏差检测通过 → 分析质量可靠
  ⚠️ 估值偏高(PE 28.5) → 分批建仓，不一次满仓

⏸️ 是否生成完整研报? (回复"确认"继续)
```

---

## 阶段5: 执行与复盘（Executor 执行智能体）

**执行步骤**:

1. **生成交易计划**（使用 templates/trade-plan.md 模板）
   - 入场条件明确
   - 止损止盈明确
   - 仓位管理明确
   - 加仓/减仓条件明确

2. **生成研报**（使用 templates/research-report.md 模板）
   - deep 模式: 完整研报（3000-5000字）
   - brief 模式: 快速简报（500-1000字）

3. **风险提示**
   - 标注所有风险因素
   - 附免责声明

4. **归档**
   - 记录本次分析的完整过程
   - 便于后续复盘对照

**输出格式**:
```
【阶段5: 执行完成】

✅ 已生成:
  📋 交易计划: 600519_交易计划_20260315
  📊 深度研报: 600519_深度分析_20260315
  
⚠️ 风险提示:
  本分析仅供参考，不构成投资建议。
  投资有风险，决策需谨慎。
  
🔄 复盘建议:
  - 1周后检查: 价格是否在预期区间
  - 1月后检查: 基本面是否有变化
  - 触发止损时: 无条件执行
```

## 特殊情况处理

### 情况1: 数据不足
```
⚠️ 数据缺失警告
- 缺失项: [具体缺失内容]
- 影响: 无法完成 [具体步骤]
- 建议: 
  1. 通过路径B执行Python脚本补充数据
  2. 或降级为brief模式（使用已有数据）
  3. 或标注"数据不足"，给出有限分析
```

### 情况2: 双框架冲突
```
⚠️ 框架冲突
- 价值框架: 建议买入（基本面优秀）
- 趋势框架: 建议观望（技术面走弱）

处理方式:
  1. 明确标注分歧点
  2. 偏保守决策（取两者的交集）
  3. 建议等待趋势信号改善后再入场
  4. 或分批建仓（先建部分价值仓位）
```

### 情况3: 风控否决
```
🚫 风控否决
- 否决原因: [具体原因，如单笔风险超限]
- 建议:
  1. 降低仓位至风控阈值内
  2. 或调整止损位
  3. 或暂时放弃该标的
```

## 响应级别

| 级别 | 触发条件 | 执行阶段 | 预估时间 |
|------|---------|---------|---------|
| **快速** | brief模式 / 单技能调用 | 阶段1+2(简化) | ~30秒 |
| **标准** | 单框架分析 | 全部5阶段 | ~2分钟 |
| **深度** | 双框架 + deep报告 | 全部5阶段(完整) | ~5分钟 |
