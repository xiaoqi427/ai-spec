# Coding 平台页面结构

## Bug 列表页

**URL**: `https://yldc.coding.yili.com/p/fssc/all/issues?filter=<filter-id>`

### 页面结构

```
┌──────────────────────────────────────────┐
│  顶部导航栏                               │
├──────────────────────────────────────────┤
│  筛选器标签栏 (我创建的/分配给我/关注的...)   │
├──────────────────────────────────────────┤
│  筛选条件区 (状态/处理人/迭代/标签等)        │
├──────────────────────────────────────────┤
│  Bug 列表                                │
│  ┌──────────────────────────────────────┐│
│  │ #ID | 标题 | 状态 | 处理人 | 优先级   ││
│  │ #4992 | xxx | 处理中 | zhangsan | 高 ││
│  │ #4991 | yyy | 待处理 | lisi | 中     ││
│  │ ...                                  ││
│  └──────────────────────────────────────┘│
│  分页器                                   │
└──────────────────────────────────────────┘
```

### 关键元素

- Bug 行项目: 通常为 `tr`, `div[class*=issue]`, `.issue-list-item`
- Bug ID: 行内包含 `#数字`
- 标题: 行内的链接文本
- 状态标签: 彩色标签元素
- 处理人: 头像 + 名字

### 注意事项

- 列表可能分页，需要滚动或翻页加载更多
- 筛选器 URL 中的 `filter` 参数决定了预设筛选条件
- 默认显示数量可能为 20/50 条

## Bug 详情页

**URL**: `https://yldc.coding.yili.com/p/fssc/bug-tracking/issues/<bug-id>/detail`

### 页面结构

```
┌──────────────────────────────────────────┐
│  面包屑导航 (项目 > 缺陷 > #<id>)         │
├──────────────────────────────────────────┤
│  左侧主区域 (约 70%)                      │
│  ┌──────────────────────────────────────┐│
│  │ 标题区                                ││
│  │ ┌──────────────────────────────────┐ ││
│  │ │ #<id> <Bug标题>                  │ ││
│  │ └──────────────────────────────────┘ ││
│  │                                      ││
│  │ 描述区                                ││
│  │ ┌──────────────────────────────────┐ ││
│  │ │ 描述正文                         │ ││
│  │ │ (可能包含图片、代码块、表格)       │ ││
│  │ └──────────────────────────────────┘ ││
│  │                                      ││
│  │ 评论区                                ││
│  │ ┌──────────────────────────────────┐ ││
│  │ │ 评论 #1: author @ time          │ ││
│  │ │ > 评论内容                       │ ││
│  │ │                                  │ ││
│  │ │ 评论 #2: author @ time          │ ││
│  │ │ > 评论内容                       │ ││
│  │ │ ...                              │ ││
│  │ └──────────────────────────────────┘ ││
│  │                                      ││
│  │ 评论输入框                             ││
│  │ ┌──────────────────────────────────┐ ││
│  │ │ 输入评论...                      │ ││
│  │ │                     [提交评论]    │ ││
│  │ └──────────────────────────────────┘ ││
│  └──────────────────────────────────────┘│
│                                          │
│  右侧属性面板 (约 30%)                    │
│  ┌──────────────────────────────────────┐│
│  │ 状态: [处理中 ▼]                     ││
│  │ 处理人: zhangsan                     ││
│  │ 优先级: 高                            ││
│  │ 迭代: Sprint-XX                      ││
│  │ 标签: 049杂项采购报账单               ││
│  │ 模块: claim-otc                      ││
│  │ 创建时间: 2025-12-15                 ││
│  │ 更新时间: 2025-12-16                 ││
│  └──────────────────────────────────────┘│
└──────────────────────────────────────────┘
```

### 关键元素

- **标题**: 页面顶部大字标题
- **描述区**: 富文本内容，可能包含 Markdown 渲染的图片、代码块
- **评论区**: 在描述区下方，按时间排列
- **评论输入框**: 评论区底部的文本输入框
- **右侧属性面板**: 状态、处理人、优先级、标签等字段
- **状态下拉框**: 点击状态标签可切换状态

### 评论区特殊说明

- 评论可能混合纯文本和富文本（Markdown）
- 评论可能包含 `@某人` 的提及
- 评论可能包含附件或截图的引用
- 活动日志和评论可能在同一区域交替显示
- 需要滚动才能看到所有评论

### 从详情页提取模板编号的策略

1. **标签字段**: "049杂项采购报账单" → 取前3位数字 → T049
2. **标题**: "T044差旅报账xxx" → 正则 `T\d{3}` → T044
3. **描述正文**: 搜索 `T\d{3}` 模式
4. **评论**: 评论中也可能提到相关模板编号

## 常见操作的 DOM 交互

### 找评论输入框

```bash
# 通常是 textarea 或 contenteditable div
browser-use state
# 在 state 输出中找类似 "输入评论" / "添加评论" / "Write a comment" 的元素
```

### 提交评论

```bash
# 找提交按钮，通常是"提交"/"发表"/"评论"按钮
browser-use state
# 在 state 输出中找类似 "提交评论" / "发表" 的按钮元素
```

### 切换状态

```bash
# 1. 点击右侧面板中的状态标签
browser-use state
# 找到当前状态元素（如 "处理中"）
browser-use click <status-idx>

# 2. 弹出状态选择列表
browser-use state
# 找到目标状态选项
browser-use click <target-status-idx>
```
