# 状态标记 & 差异等级 图例说明

> 本文件定义 compare-transform skill 输出报告中的所有标记含义
> 确保 AI 和用户对标记语义的理解一致

---

## 迁移状态标记

| 标记 | 含义 | 使用场景 |
|------|------|---------|
| ✅已迁移 | 老代码逻辑已在新代码中完整实现 | --compare, --file-migrate, --method-migrate |
| ✅基类覆盖 | 老代码逻辑已在新框架基类中实现，子类无需重写 | --file-migrate, --method-migrate |
| 📋待迁移 | 老代码逻辑尚未迁移（模式A：新文件不存在） | --file-migrate, --method-migrate |
| ⚠️有差异 | 已迁移但实现有差异，需人工确认 | --compare, --bug-check |
| ❓遗漏 | 老代码逻辑在新代码中未找到对应实现 | --compare, --bug-check, --file-migrate |
| 🔕已注释 | 老代码中该逻辑已被注释，不需要迁移 | --compare, --file-migrate, --method-migrate |
| ❌Bug | 新代码存在明确的 Bug | --bug-check |
| ✅完整 | 整个步骤迁移完整 | --compare 汇总 |
| ⚠️有疑问 | 整个步骤存在疑问需确认 | --compare 汇总 |

## 差异等级标记（--logic-diff, --method-diff）

| 标记 | 含义 | 使用场景 |
|------|------|---------|
| 🟢完全相同 | 两端代码逻辑完全一致 | --logic-diff 方法对比矩阵 |
| 🟢等价 | 写法不同但语义完全等价 | --method-diff 语句对比 |
| 🟡部分差异 | 有小部分差异，但核心逻辑相同 | --logic-diff 方法对比矩阵 |
| 🟡语义差异 | 写法不同，结果相同但语义有细微区别 | --method-diff 语句对比 |
| 🔴逻辑差异大 | 核心逻辑存在明显不同 | --logic-diff 方法对比矩阵 |
| 🔴逻辑差异 | 执行结果不同的差异 | --method-diff 语句对比 |
| 🔵仅一方有 | 只在文件A或文件B中存在 | --logic-diff, --method-diff |

---

## 重要性标记（--method-diff 特有逻辑）

| 标记 | 含义 |
|------|------|
| 🔴高 | 影响业务正确性，需立即处理 |
| 🟡中 | 可能影响边界情况，建议处理 |
| 🟢低 | 风格/优化差异，可选处理 |

---

## 模式标记（--file-migrate, --method-migrate）

| 标记 | 含义 |
|------|------|
| 📁不存在（模式A） | 新文件/方法尚未创建，输出完整迁移方案 |
| ✅已存在（模式B） | 新文件/方法已创建，检查遗漏和差异 |
