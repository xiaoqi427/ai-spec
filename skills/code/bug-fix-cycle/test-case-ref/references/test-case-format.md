# 测试用例 Excel 格式说明

## 标准列定义

| 列序 | 列名 | 字段标识 | 必填 | 说明 |
|------|------|---------|------|------|
| A | 序号 | id | 是 | 用例序号，整数 |
| B | 子功能名称(界面) | feature_name | 否 | 功能分组标识，如"报账单配置 - 查询条件"。同一分组内后续行可省略 |
| C | 用例描述 | description | 是 | 具体测试场景描述，如"验证'报账单模板'下拉可选值" |
| D | 前置条件 | precondition | 否 | 执行用例前需满足的条件 |
| E | 操作步骤 | steps | 是 | 分步操作说明，如"1. 点击下拉框；2. 查看选项" |
| F | 预期结果 | expected | 是 | 操作后的期望输出 |
| G | 实际输入 | actual_input | 否 | 测试时的实际输入数据 |
| H | 实际输出 | actual_output | 否 | 测试时的实际输出结果 |
| I | 测试人 | tester | 否 | 执行测试的人员 |
| J | 测试时间 | test_time | 否 | 测试执行日期 |
| K | 测试结论 | result | 否 | 通过 / 不通过 |
| L | 问题编号 | bug_id | 否 | 关联的 Coding Bug ID（如果测试不通过） |
| M | 测试截图\|数据 | screenshot | 否 | 截图或数据附件引用 |

---

## 解析规则

### 列头检测

- 解析器自动检测第一行作为列头
- 支持精确匹配和模糊匹配（如"子功能名称"匹配"子功能名称(界面)"）
- 列头中包含映射关键字即可识别

### 功能分组 (feature_name)

- B 列"子功能名称"用于分组，如"报账单配置 - 查询条件"、"报账单配置 - 功能按钮"
- 当 B 列为空时，沿用上一行的功能分组
- 分组名会自动清理前缀（去掉"报账单配置 - "等模块前缀）

### 空行处理

- 所有核心字段（description/steps/expected）都为空的行视为空行，跳过
- 仅有序号的行也视为空行

### 值清理

- Excel 错误值（如 #VALUE!、#REF!）自动替换为 null
- 日期值自动格式化为 YYYY-MM-DD

## 关键词提取策略

### 模板号提取

- 正则: `T\d{3}`（如 T044、T047、T051）
- 从用例描述、操作步骤、预期结果中提取

### 功能关键词提取

从以下预定义列表中匹配:

```
查询、新增、删除、修改、导出、导入、保存、提交、
重置、分页、下拉、报账单、发票、收款、付款、
审批、退回、撤回、复制、打印、校验、计算
```

### 模块名提取

从文件名中提取业务域和功能模块:

| 文件名示例 | 提取结果 |
|-----------|---------|
| 系统运维-基础数据配置-OTC销售-手工发票报账单配置.xlsx | OTC-手工发票报账单配置 |
| 系统运维-基础数据配置-OTC销售-收款方法配置.xlsx | OTC-收款方法配置 |
| 系统运维-基础数据配置-PTP采购-供应商管理.xlsx | PTP-供应商管理 |

### 业务域提取

从文件路径的父目录名提取: `doc/otc/*.xlsx` → 业务域 = "otc"

## 与 Bug ID 的关联方式

### 1. 直接关联（问题编号列）

L 列"问题编号"记录了 Coding Bug ID，可直接建立关联:

```json
{
  "bug_index": {
    "5001": ["OTC-手工发票报账单配置/查询条件/3"]
  }
}
```

### 2. 模板号匹配

Bug 标题通常包含模板号（如 "#5001 T044 金额计算错误"），通过模板号关联:

```
Bug 标题中的 "T047" → keyword_index["T047"] → ["OTC-手工发票报账单配置"]
```

### 3. 功能关键词匹配

Bug 描述中的功能词与测试用例关键词交叉匹配:

```
Bug 描述 "查询报错" → keyword_index["查询"] → ["OTC-手工发票报账单配置", "OTC-收款方法配置"]
```

```
├── otc/                     # 业务域: OTC 销售
│   ├── xxx-OTC销售-xxx.xlsx
│   └── ...
├── ptp/                     # 业务域: PTP 采购（待补充）
├── tr/                      # 业务域: TR 差旅（待补充）

```
