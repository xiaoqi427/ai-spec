## 单 Bug 处理流程

```
输入: Bug ID
  │
  ├─ 1. coding-bug-ops.read-detail(id) + read-comments(id)
  │     → 完整 Bug 上下文
  │
  ├─ 2. yili-code-fix.analyze(context)
  │     → 遗漏清单 + 前后端分类
  │
  ├─ 2a. [可选] db-query.check-data → 查库了解数据现状
  │
  ├─ 3. 展示分析方案 → ⏸️ 等待确认
  │
  ├─ 4a. [前端] → coding-bug-ops.add-comment("前端问题...")
  │
  ├─ 4b. [后端] → yili-code-fix.fix(context)
  │               → mvn compile
  │               → [可选] sit-smoke-test
  │               → coding-bug-ops.add-comment("已修复...")
  │               → [可选] coding-bug-ops.update-status
  │
  └─ 5. 输出修复报告
```

## 批量处理流程

```
输入: filter URL
  │
  ├─ 1. coding-bug-ops.read-list(url)
  │     → Bug 列表 [{id, title, status}, ...]
  │
  ├─ 2. 过滤: 只处理"待处理"/"处理中"的 Bug
  │
  ├─ 3. for each bug in list:
  │     └─ 执行"单 Bug 处理流程"
  │
  └─ 4. 输出汇总报告
```

## 状态标记说明

| 状态 | 含义 |
|------|------|
| PENDING | 等待处理 |
| ANALYZING | 正在分析 |
| DB_CHECKING | [可选] 正在查询数据库 |
| WAITING_CONFIRM | 等待用户确认 |
| FIXING | 正在修复 |
| COMPILING | 正在编译 |
| TESTING | 正在 SIT 测试 |
| COMMENTING | 正在添加评论 |
| DONE_FIX | 后端修复完成 |
| DONE_TAG | 前端标注完成 |
| SKIPPED | 用户跳过 |
| FAILED | 处理失败 |

## 错误处理策略

| 错误类型 | 处理方式 |
|----------|---------|
| 浏览器认证失败 | 降级认证策略 (Profile → Cookie → 密码) |
| Bug 详情读取失败 | 重试1次，失败则跳过并记录 |
| 老代码定位失败 | 报告给用户，等待手动指定 |
| 新代码定位失败 | 报告给用户，可能未迁移 |
| 编译失败 | 自动修复重试3次，失败则标注 |
| SIT 测试失败 | 记录失败截图，不阻塞后续 |
| 评论提交失败 | 重试1次，失败则本地保存评论内容 |
